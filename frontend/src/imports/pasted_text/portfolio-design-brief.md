Design Brief
Build a professional portfolio website for a Senior Software Engineer and DevOps specialist. The design must look like it was crafted by a thoughtful human designer who deeply understands engineering culture — not generated by AI. No purple gradients, no glowing orbs, no floating cards, no generic sans-serif stacks, no hero illustrations, no emoji bullets. If it looks like it could be on a free template site, it's wrong.
The reference points are Portez (grid clarity), Verris (black/white/gray timelessness), Metzger (stripped to essentials, work speaks for itself), and Portfolite (sophisticated restraint). Cross those with the editorial precision of Linear.app and the typographic confidence of a well-designed engineering blog. The result is dark, quiet, precise, and serious.

🚫 Hard Rules — What This Site Is NOT

No accent colors, no cyan, no green, no purple, no gradients of any kind
No hero illustrations, SVG blobs, geometric shapes, or decorative graphics
No glassmorphism, no frosted cards, no glowing borders
No typing animations, no particle effects, no scroll-triggered flying elements
No rounded pill badges for tech stack — plain text only
No icon packs (Heroicons, FontAwesome, etc.) used decoratively
No "Hi, I'm [Name] 👋" energy — no casual intros
No AI-looking layouts: asymmetric hero with a floating laptop mockup, skill bars, circular progress rings
No bold colored CTA buttons — links only
Nothing that looks like it came from a Notion template or Canva


✅ What It IS

Monochromatic dark — #0C0C0C background, pure white text, gray secondary, dark gray borders
Typography does all the heavy lifting — size, weight, and spacing create hierarchy, not color
Every element is either carrying information or creating deliberate breathing room
Feels like a senior engineer's personal site — confident, unhurried, precise
Reads like editorial design — clear structure, strong rhythm, no wasted space
Looks handcrafted, not generated

🧠 Anti-AI-Generic Checklist
Before finalising any screen, verify:

 Would a senior engineer feel proud to send this URL in a job application?
 Is every element either information or intentional space — nothing decorative?
 Does the hierarchy come from type size and weight alone — not color?
 Does it look worse without the content? (It should — the layout serves the content, not the other way around)
 Would someone mistake this for a premium human-designed site?
 Zero rounded pills, zero glow effects, zero icons used as decoration?

If any answer is no — remove or simplify until it passes.
